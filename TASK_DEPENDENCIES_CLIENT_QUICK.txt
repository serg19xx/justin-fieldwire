ЧТО ТАКОЕ ЗАВИСИМОСТИ?
─────────────────────
Зависимости — это связи между задачами, которые показывают 
последовательность выполнения работ. Система автоматически 
проверяет и контролирует эти связи.

ЗАЧЕМ ЭТО НУЖНО:
• Автоматически планирует последовательность работ
• Предотвращает ошибки в планировании  
• Показывает влияние задержек на другие задачи
• Помогает найти критический путь проекта

═══════════════════════════════════════════════════════════════

ТИПЫ ЗАВИСИМОСТЕЙ
─────────────────

FS (Finish-to-Start) - Задача B начинается после завершения A
   Пример: Фундамент → Стены

SS (Start-to-Start) - Задача B начинается одновременно с A  
   Пример: Подготовка участка → Установка оборудования

FF (Finish-to-Finish) - Задача B завершается одновременно с A
   Пример: Электрика ↔ Сантехника

SF (Start-to-Finish) - Задача B завершается при начале A
   Пример: Новая система → Отключение старой

═══════════════════════════════════════════════════════════════

ЛАГИ (ЗАДЕРЖКИ)
───────────────
Лаг — дополнительное время между связанными задачами.

Пример:
Задача A: "Заливка фундамента" (1-5 января)
Задача B: "Кладка стен" (8-15 января)  
Лаг: 2 дня — время на застывание бетона

═══════════════════════════════════════════════════════════════

КАК РАБОТАЕТ В КАЛЕНДАРЕ
────────────────────────

1. ПЕРЕМЕЩЕНИЕ ЗАДАЧ:
   • Перетащите задачу на новую дату
   • Система проверит зависимости автоматически
   • При нарушениях появится диалог с предупреждением
   • Выберите: "Cancel" (отменить) или "Adjust" (применить)

2. КАСКАДНЫЕ ИЗМЕНЕНИЯ:
   При изменении одной задачи система предлагает изменения 
   для связанных задач.

   Пример каскада:
   1. Сдвигаете "Фундамент" на 3 дня вперед
   2. Система предлагает сдвинуть "Стены" на 3 дня вперед  
   3. Затем предлагает сдвинуть "Крыша" на 3 дня вперед
   4. И так далее по цепочке зависимостей

ВАЖНЫЕ ОГРАНИЧЕНИЯ
──────────────────
❌ НЕЛЬЗЯ:
• Создавать циклические зависимости (A→B→A)
• Перемещать задачи за границы проекта  
• Нарушать логические связи между задачами

═══════════════════════════════════════════════════════════════

ПРАКТИЧЕСКИЕ СОВЕТЫ
───────────────────

ДЛЯ ПЛАНИРОВАНИЯ:
• Начните с основных задач без зависимостей
• Добавляйте зависимости постепенно
• Используйте FS для последовательных работ (самый частый)
• Устанавливайте реалистичные лаги для буферного времени

ПРИ РАБОТЕ С КАЛЕНДАРЕМ:
• Читайте предупреждения системы внимательно
• Используйте "Adjust" для автоматической корректировки
• Проверяйте каскадные изменения перед подтверждением
• При сомнениях — отменяйте изменения и планируйте заново
