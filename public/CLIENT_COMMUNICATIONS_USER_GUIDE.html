Clients: SMS, Email & Fax

FieldWire — send messages to client registry contacts (Admin & Project Manager)

Send operational SMS, email, and fax to pharmacies, physicians, pharmacists, and medical clinics from the Clients menu — the same workflow as legacy EasyRx client messaging, integrated into FieldWire.

Who can use this

RoleAccess
AdministratorFull access
Project ManagerFull access
Other rolesClients menu and send actions are not available

Where to find it

  1. Sign in to FieldWire.
  2. Open Clients in the main navigation.
  3. Choose a list: Pharmacies, Physicians & Providers, Pharmacists, or Medical clinics.
  4. Use row action icons (SMS, email, fax) or multi-select rows for bulk SMS/email.

Channels overview

ChannelListsBulkLimits
SMSPharmacies, Physicians, PharmacistsYesPlain text, max 1600 characters
EmailAll four listsYesFree-form or SendGrid dynamic template
FaxPharmacies, Physicians, Medical clinicsNoPDF/image up to 50 MB + optional cover page

Pharmacists support SMS and email only (no fax column).

SMS

  1. Click the green SMS icon on a row.
  2. Type your message (character counter; multiple segments if long).
  3. Click Send SMS.

Bulk: select two or more rows → Send SMS in the selection bar. Rows without a phone number are skipped.

Email

  1. Click the email icon on a row.
  2. Choose Email format:
    • Free form letter — your subject and body.
    • SendGrid template — layout from SendGrid; client name is passed automatically.
  3. Click Send Email.

Bulk: multi-select → Send Email. Rows without email are skipped.

Fax

  1. Click the fax icon on a row.
  2. Optionally enter Subject and Cover text.
  3. Attach a document: PDF or image (JPEG, PNG, GIF, TIFF, BMP, WebP), max 50 MB.
  4. Click Send Fax. Large files may take up to about one minute.
You need cover text or an attachment (or both). Bulk fax is not supported.

Contact filter

Use Filter: Phone / Email / Fax with Has value or Empty to find clients reachable on each channel.

Export CSV

Export CSV downloads the filtered list. Load from CSV is planned but not available yet.

How it is built (technical overview)

The browser calls the FieldWire API; the API sends through server-side providers (credentials never appear in the frontend):

ChannelProvider
EmailSendGrid (API + optional dynamic templates)
SMSTwilio
FaxHumbleFax (cover sheet + document upload, then queue)

Flow for fax: create draft on HumbleFax → upload attachment → send to recipient fax number from the client record.
