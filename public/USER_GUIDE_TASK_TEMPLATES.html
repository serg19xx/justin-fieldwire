Task templates — user guide

Where in the app: Settings → Task templates

Who can use it: Admin and Project Manager.

What is this?

A task template is a ready-made recipe for a task.

Instead of typing the same task again and again (“Install cabinets”, “Rough-in plumbing”…), you save it once as a template. Later, when you build a project schedule, you pick the template and the system fills in the usual fields.

Simple idea: Template = blank form already filled with defaults. Creating a template does not create a real task in a project yet.

How to open it

  1. Log in as Admin or Project Manager.
  2. Settings → Task templates.

What you see

  • A list of templates (name, category, and actions)
  • Filters: category and search
  • Button Add Template

Create a template

  1. Click Add Template.
  2. Enter a clear name (example: Paint interior walls).
  3. Choose a category if asked (helps you find it later).
  4. Fill the fields you usually set on a real task (description, defaults, etc.).
  5. Save.

After save, the template appears in the list.

Edit or delete

ActionWhat happens
EditChange the recipe. New projects/tasks created from it will use the new values. Old tasks already created stay as they are.
DeleteRemoves the template from the list. Does not delete tasks that were already created from it.

How templates are used in a project

When you add tasks inside a project (calendar / task list / importer — depending on the screen), you can choose a template instead of starting from an empty form.

Typical flow:

  1. Open a project.
  2. Add task → choose template (or import from templates).
  3. Adjust dates/people for this project.
  4. Save the real task.

Tips

  • Use short, clear names.
  • Group with categories (Electrical, Plumbing, Finish…).
  • Do not put project-specific dates in the template — those change every project.

How to test

  1. Create a test template named TEST – Demo task.
  2. Find it with Search.
  3. Edit the description and save.
  4. Open a test project and create a task from that template.
  5. Confirm the task got the expected fields.
  6. Delete the test template when finished.
