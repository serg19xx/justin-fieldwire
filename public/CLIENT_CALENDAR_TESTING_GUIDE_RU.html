Руководство для тестирования: личный календарь

FieldWire · версия 1.0 · fieldwire.medicalcontractor.ca

Важно: это не календарь задач. В разделе Tasks календарный вид отключён. Новый Calendar — отдельный личный календарь (встречи, напоминания, визиты). События не связаны с задачами проекта.

1. Два режима календаря

Глобальный календарь Календарь проекта
Где открыть Пункт Calendar в верхнем меню Проект → боковое меню Calendar (между Schedule и Photos)
Что видно Все ваши события: личные + из всех проектов Личные + события текущего проекта
Что редактируется Только личные события Только события этого проекта
Другие проекты Видны, только просмотр Скрыты
Личные события Синие, редактируемые Светло-синие, только просмотр
Правило: личное событие создавайте и меняйте в глобальном календаре. Событие проекта — во вкладке Calendar внутри нужного проекта.

2. Типы событий

В форме события есть чекбокс Requires my presence («Требуется моё присутствие»).

Тип Чекбокс Поведение
Reminder (напоминание) выключен Можно создать сколько угодно событий на одно время. Конфликты не проверяются.
Requires my presence включён Проверка пересечений с другими presence-событиями этого пользователя (личный календарь + все проекты).

При конфликте presence

  1. Первое нажатие Save — показывается список занятых слотов.
  2. Save anyway — сохранить несмотря на пересечение.
  3. Cancel — отменить.

Напоминания (Reminder) никогда не блокируют сохранение и не участвуют в проверке чужих presence-событий.

3. Цвета и легенда

Глобальный календарь

  • Синий — личное (редактируемое)
  • Серый — проект (только просмотр)
  • Янтарная рамка — Requires my presence

Календарь проекта

  • Зелёный — проект (редактируемое)
  • Светло-синий — личное (только просмотр)
  • Янтарная рамка — Requires my presence

4. Как создать событие

Месяц (Month)

  • Один клик по дню — выделение даты (диалог не открывается).
  • Двойной клик или кнопка + New event — форма создания.
  • По умолчанию для события с временем: 10:00–12:00.

Неделя / День (Week / Day)

  • Перетащите мышью интервал времени.
  • Или двойной клик по ячейке времени.

Поля формы

  • Title — обязательно
  • Requires my presence — тип события
  • Notes, Location — по желанию
  • All day — целый день
  • Даты и время начала/окончания

5. Сценарии тестирования

Тест 1 — Личное напоминание (глобальный календарь)

  1. Меню Calendar → + New event.
  2. Заполнить заголовок и время. Чекбокс presence — выключен.
  3. Save. Ожидание: синее событие, можно редактировать и удалить.

Тест 2 — Два напоминания на одно время

  1. Создать два Reminder на одинаковое время.
  2. Ожидание: оба сохраняются без предупреждения.

Тест 3 — Presence и конфликт

  1. Событие A: presence, например Пн 10:00–12:00.
  2. Событие B на пересекающееся время, тоже с presence.
  3. Ожидание: список конфликтов, кнопка Save anyway.

Тест 4 — Конфликт между проектом и личным календарём

  1. В проекте A создать presence на время T.
  2. В глобальном календаре создать presence на то же T.
  3. Ожидание: конфликт. Reminder на T — без конфликта.

Тест 5 — Права редактирования

  1. Событие проекта в глобальном календаре — серое, только просмотр.
  2. Редактировать его в Calendar внутри того же проекта.
  3. Личное событие в проекте — только просмотр; менять в глобальном календаре.

Тест 6 — События другого проекта

  1. Событие в проекте B.
  2. В календаре проекта A — не видно.
  3. В глобальном календаре — видно (серое).

6. Ограничения

  • Нет синхронизации с Google Calendar / Outlook.
  • События не создают и не меняют задачи в Tasks.
  • Проверка presence — только для текущего пользователя.
  • Save anyway — осознанный обход; жёсткой блокировки двойного бронирования нет.

7. Если что-то не работает

Симптом Что проверить
Ошибка 500 при сохранении Сообщить разработчику (серверная настройка БД).
Нет пункта Calendar Обновить страницу (Ctrl+F5).
Нет конфликта У обоих событий включён Requires my presence; время пересекается.
Не редактируется Событие проекта — только в Calendar этого проекта.
