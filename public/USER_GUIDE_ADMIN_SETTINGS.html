Admin settings — user guide

Where in the app: Settings → Admin settings

Who can use it: mainly Admin (and Project Manager if allowed).

What is this?

This is the control panel for automatic messages and rules.

There are two tabs:

  1. Event Rules — when something happens, what should the system do?
  2. Message Templates — ready-made texts for email/SMS.
Simple idea:
Rule = “IF this happens, THEN do that.”
Template = “Here is the text to send.”

1) Event Rules

What a rule does

A rule watches for an event type (example: project status changed, weekly report time).

When that event happens, the rule can:

  • Notify — send email / SMS / push to selected roles
  • Create Report — build a report and send it
  • Log Only — only write to the Event log (no message)

Main fields (plain words)

FieldMeaning
Event TypeThe name of the thing we listen for (usually UPPERCASE_WITH_UNDERSCORES)
Rule EnabledOn = rule works. Off = ignored
SeverityHow important it looks in the log (Important / Critical)
PriorityHow urgently the system processes the job queue. Leave Auto if unsure
Execution LocationWhere it runs. Leave Auto/Server if unsure
ActionsWhat to do (Notify / Report / Log Only)
ScheduleOptional timer. Empty = run right away when the event happens

Notify action

  • Choose channels: EMAIL, SMS, PUSH
  • Choose who gets it (Admin, Project Manager, Foreman…)
  • Choose message source:
    • System text — automatic short text
    • Message Template — pick a template you created
    • Custom text — type the text on the rule

Email goes through SendGrid. SMS goes through Twilio. They only deliver — they do not invent the wording.

Create Report action

  • Period = what the report covers (daily / weekly / monthly data)
  • Recipients = who gets the report email
  • When it runs is controlled by Schedule (not by Period)

Schedule (optional)

Like a calendar alarm:

  • Frequency — daily / weekly / monthly
  • Days / day of month / 2nd Monday… — which day
  • At time and Until — time window when it is allowed to run
  • Timezone — which clock to use (example: America/Toronto)

Why a window (At → Until)? The system checks every few minutes. A short window (07:00–07:30) avoids missing an exact minute.

  • No schedule → process immediately when the event happens
  • With schedule → wait for the right day/time window

Click the small ? icons next to fields for short help.

How to create a simple notify rule (test)

  1. Open Admin settings → Event Rules.
  2. Create / edit a rule.
  3. Set Event Type (use an existing type from your system, or a known test type).
  4. Enable the rule.
  5. Add action Notify → EMAIL → choose recipients → System text or a template.
  6. Leave Schedule empty for “immediate”.
  7. Save.
  8. Trigger the real action in the app, then check email and Event log.

2) Message Templates

What a template is

Reusable text for email or SMS. You write it once, then Event Rules can use it.

How to create one

  1. Open tab Message Templates.
  2. Click Create Template.
  3. Name it clearly (example: Task assigned SMS).
  4. Choose type: Email or SMS.
  5. For Email, fill Subject.
  6. Write the body. Insert variables with buttons like {{PROJECT_NAME}}.
  7. In Variables, type sample values to see Preview.
  8. Keep Active checked.
  9. Save.

Variables

A variable is a placeholder. Example:

Hello, work on {{PROJECT_NAME}} starts tomorrow.

When the message is sent for a real project, {{PROJECT_NAME}} becomes the real name.

Preview uses your sample values so you can check the text looks right.

How to test a template

  1. Create a template with a few variables.
  2. Open Preview in the editor — confirm sample values appear.
  3. From the list, click Preview — same check.
  4. Attach the template to an Event Rule Notify action (Source = Message Template).
  5. Trigger the event and confirm the real message arrives.

How the pieces fit together

  1. Something happens in the app (or a schedule timer fires).
  2. An Event Rule matches.
  3. If Schedule allows it, the system creates a job.
  4. Notify uses a Message Template (or system/custom text).
  5. You can later inspect what happened in Event log.
  6. Reports may appear under Settings → Reports.
Be careful: a wrong rule can send many emails/SMS. Test with yourself as recipient first. Turn the rule off when not ready.

Quick checklist for the customer

  1. Create one SMS template and one Email template. Preview works.
  2. Create one Notify rule using that template. Recipients = Admin only.
  3. Trigger a test event. Message arrives. Event log shows the event.
  4. (Optional) Add a Schedule window and confirm it only fires inside that window.
  5. (Optional) Create a Create Report rule with monthly schedule on day 1.
