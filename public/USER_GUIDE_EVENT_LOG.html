Event log — user guide

Where in the app: Settings → Event log

Who can use it: Admin and Project Manager.

What is this?

A diary of important things the system recorded.

Examples:

  • a project became active
  • a task status changed
  • someone got a notification
  • a report was generated

You use Event log to answer: “What happened, when, and about which project?”

Simple idea: Event log is history. It does not create work by itself. Rules that send emails/SMS live in Admin settings.

How to open it

  1. Log in.
  2. Open the Settings menu (gear).
  3. Click Event log.

What you see on the page

  • A list of events (newest first, usually)
  • Filters at the top so you can find one event faster
  • A Refresh button to reload the list

How to find something

FilterWhat it does
SearchType a word from a comment, event name, or project
Event typeOnly one kind of event (example: task changed)
ProjectOnly events for one project
From / ToOnly events between two dates

After you set filters, apply them (button or Enter in search) and wait for the list to update.

How to read one event

Click a row (or open details if the UI shows an expand/open control). Look for:

  • When it happened
  • What type of event
  • Which project or task
  • Who did it (user or system)
  • Any short comment

What Event log is NOT

  • It is not a chat
  • It is not where you turn notifications on/off (that is Admin settings → Event Rules)
  • It is not where you write message text (that is Message Templates)

How to test

  1. Open Event log — confirm the page loads.
  2. Do a small action in the app (example: change a task status in a test project).
  3. Come back to Event log → Refresh.
  4. Find your event with filters (project + date today).
  5. Open details and confirm the information looks correct.

If the list is empty

  • Widen the date range
  • Clear filters
  • Make sure something actually happened in the system recently
  • Confirm Event Rules exist and are enabled (Admin settings) if you expected automatic events
