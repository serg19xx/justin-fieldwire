## Введение

FieldWire - это система управления строительными проектами, предназначенная для сопровождения и поддержки строительных проектов. Данное руководство предназначено для пользователей с ролью **System Administrator** (Системный администратор).

### Роль администратора

Системный администратор имеет полный доступ ко всем функциям приложения:
- Управление всеми проектами в системе
- Управление всеми пользователями и командой
- Настройка системных правил и шаблонов сообщений
- Просмотр всех отчетов и аналитики
- Назначение менеджеров проектов

## Вход в систему

### Авторизация

1. Откройте приложение FieldWire в браузере
2. На странице входа введите:
   - **Email** - ваш адрес электронной почты
   - **Password** - ваш пароль
3. Нажмите кнопку **"Sign In"**

### Сброс пароля

Если вы забыли пароль:
1. На странице входа нажмите **"Forgot password?"**
2. Введите ваш email
3. Следуйте инструкциям в письме для сброса пароля

### Смена пароля

Если у вас временный пароль:
1. После входа в систему вы будете перенаправлены на страницу смены пароля
2. Введите текущий пароль
3. Введите новый пароль (должен соответствовать требованиям безопасности)
4. Подтвердите новый пароль
5. Нажмите **"Change Password"**

## Управление проектами

### Список проектов

Доступ к списку всех проектов через меню **Projects**.

#### Функции списка проектов:

1. **Поиск проектов**
   - Используйте поле поиска для фильтрации проектов по названию
   - Поиск работает в реальном времени

2. **Фильтрация проектов**
   - По статусу (Active, Completed, On Hold, Draft)
   - По приоритету (High, Medium, Low)
   - По менеджеру проекта

3. **Сортировка проектов**
   - По названию (A-Z, Z-A)
   - По дате создания
   - По статусу
   - По прогрессу

4. **Статистика проектов**
   - Общее количество проектов
   - Активные проекты
   - Завершенные проекты
   - Проекты на паузе

### Создание нового проекта

1. На странице списка проектов нажмите кнопку **"New Project"**
2. Заполните форму создания проекта:
   - **Project Name** * (обязательно) - Название проекта
   - **Address** - Адрес проекта
   - **Description** - Описание проекта
   - **Priority** - Приоритет (High, Medium, Low)
   - **Status** - Статус (Draft, Active, On Hold, Completed)
   - **Start Date** - Дата начала проекта
   - **End Date** - Дата окончания проекта
   - **Project Manager** - Менеджер проекта (только администратор может назначать)
3. Нажмите **"Create Project"**

### Просмотр проекта (Admin View)

Администраторы имеют специальный вид проекта (`/projects/:id/admin`), который предоставляет:
- Обзор проекта
- Статистика по задачам
- Информация о команде
- Быстрый доступ к настройкам

### Редактирование проекта

1. Откройте проект из списка
2. Перейдите в раздел **Settings**
3. Измените необходимые поля:
   - Название проекта
   - Адрес
   - Описание
   - Приоритет
   - Статус
   - Даты начала и окончания
   - Менеджер проекта
4. Нажмите **"Save Changes"**

### Удаление проекта

1. Откройте проект
2. Перейдите в раздел **Settings**
3. Нажмите кнопку **"Delete Project"**
4. Подтвердите удаление

## Управление командой (Builders)

### Доступ к управлению командой

Меню **Builders** предоставляет доступ к управлению всеми пользователями системы.

### Режимы просмотра

1. **Registered** - Зарегистрированные пользователи
2. **Pending** - Пользователи с ожидающими приглашениями

### Функции управления командой

#### Поиск и фильтрация

1. **Поиск по тексту**
   - Введите имя, фамилию, email или должность в поле поиска
   - Поиск работает в реальном времени

2. **Фильтры**
   - **User Type** - Фильтр по типу пользователя (роли)
   - **Status** - Фильтр по статусу (Active, Inactive)
   - **Invitation Status** - Фильтр по статусу приглашения (Registered, Invited)

#### Просмотр информации о пользователе

1. Кликните на карточку пользователя
2. Откроется диалог с подробной информацией:
   - Личная информация
   - Контактные данные
   - Роль и права доступа
   - Статус в системе
   - История активности

#### Приглашение нового пользователя

1. Нажмите кнопку **"Invite Builder"**
2. Заполните форму приглашения:
   - **Email** * - Email адрес пользователя
   - **First Name** * - Имя
   - **Last Name** * - Фамилия
   - **Role** - Роль в системе
   - **Job Title** - Должность
   - **Phone** - Телефон
3. Нажмите **"Send Invitation"**

#### Редактирование пользователя

1. Откройте карточку пользователя
2. Нажмите кнопку **"Edit"**
3. Измените необходимые поля
4. Нажмите **"Save Changes"**

#### Удаление пользователя

1. Откройте карточку пользователя
2. Нажмите кнопку **"Delete"**
3. Подтвердите удаление

## Административные настройки

### Доступ к административным настройкам

Меню **Admin Settings** доступно только для пользователей с ролью **System Administrator**.

### Разделы административных настроек

#### 1. Event Rules (Правила событий)

Правила событий позволяют настроить автоматические действия при возникновении определенных событий в системе.

##### Просмотр правил

1. Откройте раздел **Event Rules**
2. Просмотрите список всех правил:
   - **System Rules** - Системные правила (созданные системой)
   - **Custom Rules** - Пользовательские правила (созданные администраторами)

##### Фильтрация правил

- По типу правила (System/Custom)
- По типу события (Event Type)
- По статусу (Enabled/Disabled)
- По месту выполнения (Execution Location)

##### Создание нового правила

1. Нажмите кнопку **"Create Rule"**
2. Заполните форму:
   - **Event Type** * - Тип события
   - **Rule Name** * - Название правила
   - **Description** - Описание правила
   - **Enabled** - Включено/Выключено
   - **Execution Location** - Место выполнения (Frontend/Backend)
   - **Conditions** - Условия срабатывания правила
   - **Actions** - Действия при срабатывании правила
3. Нажмите **"Create Rule"**

##### Редактирование правила

1. Найдите правило в списке
2. Нажмите кнопку **"Edit"**
3. Измените необходимые поля
4. Нажмите **"Update Rule"**

##### Удаление правила

**Важно:** Удалить можно только пользовательские правила (Custom Rules). Системные правила удалить нельзя.

1. Найдите пользовательское правило в списке
2. Нажмите кнопку **"Delete"**
3. Подтвердите удаление

#### 2. Message Templates (Шаблоны сообщений)

Шаблоны сообщений используются для автоматической отправки уведомлений.

##### Типы шаблонов

- **Email Templates** - Шаблоны email сообщений
- **SMS Templates** - Шаблоны SMS сообщений

##### Просмотр шаблонов

1. Откройте раздел **Message Templates**
2. Выберите тип шаблона (Email/SMS)
3. Просмотрите список шаблонов

##### Создание нового шаблона

1. Нажмите кнопку **"Create Template"**
2. Заполните форму:
   - **Template Name** * - Название шаблона
   - **Subject** - Тема сообщения (для email)
   - **Body** * - Текст шаблона
   - **Variables** - Доступные переменные для подстановки
3. Нажмите **"Create Template"**

##### Редактирование шаблона

1. Найдите шаблон в списке
2. Нажмите кнопку **"Edit"**
3. Измените необходимые поля
4. Нажмите **"Update Template"**

##### Удаление шаблона

1. Найдите шаблон в списке
2. Нажмите кнопку **"Delete"**
3. Подтвердите удаление

## Работа с проектами (детальный обзор)

### Структура проекта

При открытии проекта администратор видит следующие разделы:

1. **Plans** - Планы и чертежи
2. **Tasks** - Задачи проекта
3. **Photos** - Фотографии проекта
4. **Team** - Команда проекта
5. **Settings** - Настройки проекта

### Раздел Plans (Планы)

#### Управление файлами

1. **Загрузка файлов**
   - Нажмите кнопку **"Upload"**
   - Выберите файлы для загрузки
   - Выберите папку назначения
   - Нажмите **"Upload"**

2. **Создание папок**
   - Нажмите кнопку **"New Folder"**
   - Введите название папки
   - Нажмите **"Create"**

3. **Просмотр файлов**
   - Режим иконок (Icons View)
   - Режим списка (Details View)

4. **Операции с файлами**
   - **Preview** - Просмотр файла
   - **Download** - Скачать файл
   - **Rename** - Переименовать
   - **Move** - Переместить в другую папку
   - **Delete** - Удалить файл

### Раздел Tasks (Задачи)

#### Просмотр задач

Задачи отображаются в нескольких представлениях:
- **Calendar View** - Календарное представление
- **Gantt View** - Диаграмма Ганта
- **List View** - Список задач

#### Создание задачи

1. Нажмите кнопку **"New Task"** или кликните на дату в календаре
2. Заполните форму:
   - **Task Name** * - Название задачи
   - **WBS Path** - Путь в структуре работ
   - **Start Date** * - Дата начала
   - **End Date** * - Дата окончания
   - **Duration** - Длительность (дни)
   - **Status** - Статус (Planned, In Progress, Done, Blocked, Delayed)
   - **Progress** - Прогресс выполнения (%)
   - **Task Lead** - Ответственный
   - **Team Members** - Члены команды
   - **Dependencies** - Зависимости от других задач
   - **Resources** - Ресурсы
   - **Notes** - Заметки
3. Нажмите **"Create Task"**

#### Создание майлстоуна

1. Нажмите кнопку **"New Milestone"** или выберите опцию при создании задачи
2. Заполните форму:
   - **Milestone Name** * - Название майлстоуна
   - **Milestone Type** - Тип (Inspection, Visit, Meeting, Review, Delivery, Approval, Other)
   - **Date** * - Дата майлстоуна
   - **Status** - Статус
   - **Project Lead** * - Ответственный
   - **Invited People** - Приглашенные люди (внешние участники)
   - **Notes** - Заметки
3. Нажмите **"Create Milestone"**

#### Редактирование задачи

1. Кликните на задачу в календаре или списке
2. Откроется диалог редактирования
3. Измените необходимые поля
4. Нажмите **"Save Changes"**

#### Редактирование майлстоуна

1. Кликните на майлстоун в календаре
2. Откроется диалог редактирования майлстоуна
3. Измените необходимые поля
4. Нажмите **"Save Changes"**

#### Удаление задачи/майлстоуна

1. Откройте задачу/майлстоун для редактирования
2. Нажмите кнопку **"Delete"**
3. Подтвердите удаление

#### Дублирование задачи

1. Откройте задачу для редактирования
2. Нажмите кнопку **"Duplicate"**
3. Задача будет скопирована с новым названием

#### Зависимости задач

1. Откройте задачу для редактирования
2. Перейдите в раздел **Dependencies**
3. Нажмите **"Add Dependency"**
4. Выберите задачу-предшественника
5. Выберите тип зависимости:
   - **FS (Finish-to-Start)** - Окончание-начало
   - **SS (Start-to-Start)** - Начало-начало
   - **FF (Finish-to-Finish)** - Окончание-окончание
   - **SF (Start-to-Finish)** - Начало-окончание
6. Укажите задержку (lag days)
7. Нажмите **"Add"**

#### Назначение команды на задачу

1. Откройте задачу для редактирования
2. Перейдите в раздел **Team & Resources**
3. Выберите **Task Lead** - ответственного за задачу
4. Выберите **Team Members** - членов команды для выполнения задачи
5. Нажмите **"Save Changes"**

#### Экспорт задач в iCal

1. Откройте раздел **Tasks**
2. Нажмите кнопку **"Export to iCal"**
3. Файл будет скачан в формате .ics

### Раздел Photos (Фотографии)

#### Загрузка фотографий

1. Нажмите кнопку **"Upload Photo"**
2. Выберите фотографии
3. Добавьте описание (опционально)
4. Нажмите **"Upload"**

#### Просмотр фотографий

- Фотографии отображаются в виде галереи
- Кликните на фотографию для просмотра в полном размере

### Раздел Team (Команда)

#### Просмотр команды проекта

В разделе Team отображается список всех членов команды проекта с информацией:
- Имя и фото
- Роль в проекте
- Назначенные задачи
- Статус

#### Добавление члена команды

1. Нажмите кнопку **"Add Team Member"**
2. Выберите пользователя из списка доступных
3. Выберите роль в проекте
4. Нажмите **"Add"**

#### Просмотр деталей члена команды

1. Кликните на карточку члена команды
2. Откроется диалог с подробной информацией:
   - Личная информация
   - Роль в проекте
   - Назначенные задачи
   - История активности

#### Удаление члена команды

1. Откройте детали члена команды
2. Нажмите кнопку **"Remove from Project"**
3. Подтвердите удаление

#### Назначение работника на задачу

1. В разделе Team найдите задачу
2. Нажмите кнопку **"Assign Worker"**
3. Выберите работника из списка
4. Нажмите **"Assign"**

#### Удаление работника с задачи

1. В разделе Team найдите задачу
2. Найдите назначенного работника
3. Нажмите кнопку **"Remove"** рядом с работником
4. Подтвердите удаление

### Раздел Settings (Настройки)

#### Основные настройки проекта

- **Project Name** - Название проекта
- **Address** - Адрес проекта
- **Description** - Описание проекта
- **Priority** - Приоритет (High, Medium, Low)
- **Status** - Статус (Draft, Active, On Hold, Completed)
- **Start Date** - Дата начала проекта
- **End Date** - Дата окончания проекта
- **Project Manager** - Менеджер проекта (только администратор может изменять)

#### Сохранение изменений

1. Измените необходимые поля
2. Нажмите кнопку **"Save Changes"**
3. Изменения будут сохранены

## Советы и рекомендации

### Безопасность

- Регулярно меняйте пароль
- Используйте двухфакторную аутентификацию
- Не передавайте свои учетные данные другим лицам

### Производительность

- Используйте фильтры для быстрого поиска проектов и задач
- Экспортируйте данные для анализа вне системы
- Регулярно проверяйте и очищайте неактивные проекты

### Лучшие практики

- Всегда заполняйте описание проекта при создании
- Устанавливайте реалистичные даты начала и окончания проектов
- Назначайте ответственных за каждую задачу
- Регулярно обновляйте статусы задач
- Используйте майлстоуны для отслеживания ключевых событий
