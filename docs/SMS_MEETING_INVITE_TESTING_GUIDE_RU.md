Документ для заказчика и QA. Описывает, **что реализовано**, **как пользоваться** и **как проверить** функцию **Schedule call via SMS** в production.

## 1. Что сделано (кратко для заказчика)

| # | Функция | Описание |
|---|---------|----------|
| 1 | **Schedule call via SMS** | PM из списка Clients отправляет клиенту SMS с вариантами времени звонка |
| 2 | **Выбор даты PM** | PM сам выбирает дату; минимум **3 дня** от сегодня, **без ограничения неделей** |
| 3 | **Расписание дня** | Сетка **30 мин**, **9:00–17:00**; занятые окна помечены как недоступные |
| 4 | **Выбор слотов PM** | PM отмечает **от 1 до 3** свободных слотов на выбранную дату |
| 5 | **Удержание слотов** | Выбранные слоты **зарезервированы 3 дня** до ответа клиента |
| 6 | **Понятное SMS** | Клиент видит имя и роль PM, тему, дату, длительность и варианты времени |
| 7 | **Ответ 1 / 2 / 3** | Клиент отвечает SMS с телефона; при 1–2 слотах — «1» или «1 or 2» |
| 8 | **Calendar** | После подтверждения — событие в календаре **отправившего PM** |
| 9 | **Статус в диалоге** | Pending / Confirmed / Expired для последнего инвайта клиента |

**Связанные функции (уже работают):** SMS, Email, Fax из Clients — см. [руководство по коммуникациям](https://fieldwire.medicalcontractor.ca/CLIENT_COMMUNICATIONS_USER_GUIDE_RU.html).

## 3. Подготовка к тестированию

### 3.1. Окружение

- **URL:** https://fieldwire.medicalcontractor.ca
- **API:** https://fwapi.medicalcontractor.ca (работает в фоне, отдельно не открывается)
- **Режим:** production — SMS уходят на **реальные номера** из карточек клиентов

### 3.2. Что нужно иметь

- [ ] Учётная запись **Admin** или **Project Manager**
- [ ] Клиент в списке **Clients** с **телефоном** (cell / office / phone)
- [ ] Доступ к **мобильному телефону** клиента (или тестовому номеру) для чтения и ответа SMS
- [ ] Свободные окна в **Calendar** PM на выбранную дату (или намеренно занятые — для проверки сетки)

### 3.3. Рекомендуемая подготовка календаря PM

Для сценария с сеткой слотов создайте на тестовую дату (не раньше чем через 3 дня):

- одно событие, например **11:00–11:30** — чтобы слот 11:00 стал недоступен;
- оставьте свободными минимум **3 полчасовых окна** для выбора.

## 4. Как пользоваться (инструкция PM)

1. Войти в FieldWire → **Clients** → список с клиентами (например, **Physicians & Providers**).
2. Найти клиента с телефоном (фильтр **Phone → Has value**).
3. Действие в строке: **Schedule call via SMS**.
4. **Meeting title** — при необходимости изменить (по умолчанию «Call with …»).
5. **Meeting date** — выбрать дату (**не раньше чем через 3 дня**).
6. Дождаться загрузки **Day schedule** — сетка 9:00–17:00, шаг 30 мин.
7. Кликнуть **1–3 свободных** (зелёные/белые) слота; занятые (серыe) недоступны.
8. Проверить **SMS preview**.
9. **Send meeting invite**.
10. Клиент отвечает **1**, **2** или **3** (или **1** / **1 or 2**, если PM выбрал меньше слотов).
11. Проверить **Calendar** PM — появилось событие.
12. Снова открыть диалог для клиента — статус **Confirmed**.

**Удержание:** выбранные слоты помечены как занятые **3 дня** (другие инвайты PM их не увидят как свободные).

## 6. Пример SMS (для сверки)

**Инвайт (3 слота):**

```
FieldWire — meeting request

Hi Dr. Smith,

Jane Doe (Project Manager) would like to schedule a call with you.

Topic: Call with Dr. Smith
Date: Jun 25, 2026
Duration: 30 min

Reply with your preferred time:
1) 10 AM
2) 2 PM
3) 3:30 PM

Text back 1, 2, or 3.
```

**Подтверждение (ответ «2»):**

```
FieldWire — confirmed

Thank you, Dr. Smith!

Call with Dr. Smith
Jun 25, 2026 at 2 PM

— Jane Doe, Project Manager
```

## 7. Решение проблем

| Проблема | Что проверить |
|----------|----------------|
| Failed to send SMS | Телефон в карточке; Twilio; корректность номера |
| Нет свободных слотов на дату | Calendar PM; другие pending-инвайты; выбрать другую дату |
| SMS пришло, ответ не работает | Ответ с **того же телефона**; только цифры 1–3 |
| Нет события в Calendar | Тот же PM, что отправил; обновить Calendar |
| No active meeting request | Прошло **3 дня** или новый инвайт заменил старый |
| Дата не выбирается | Минимум **+3 дня** от сегодня |
