# Руководство: запись на звонок через SMS (FieldWire)

Project Manager отправляет клиенту SMS с тремя вариантами времени; клиент отвечает **1**, **2** или **3** — в календаре PM автоматически создаётся событие.

## Как открыть это руководство

- **Веб (после деплоя):** https://fieldwire.medicalcontractor.ca/SMS_MEETING_INVITE_USER_GUIDE_RU.html
- **English:** https://fieldwire.medicalcontractor.ca/SMS_MEETING_INVITE_USER_GUIDE.html
- **Из приложения:** Clients → любой список → ссылка **Schedule call via SMS guide** в шапке страницы.

## Что реализовано и зачем

| Возможность | Назначение |
|-------------|------------|
| **Schedule call via SMS** в списке Clients | PM быстро предлагает клиенту 3 слота для звонка без звонков и переписки вручную |
| **Понятное SMS** | Клиент видит, кто пишет (имя и роль PM), тему, дату, длительность и три времени |
| **Ответ 1 / 2 / 3** | Клиент выбирает слот обычным SMS-ответом с телефона |
| **Автоматический календарь** | После подтверждения у PM появляется личное событие в **Calendar** |
| **Статус в диалоге** | PM видит, ожидается ли ответ, подтверждён ли слот или истёк ли срок |

Это дополнение к уже работающим **SMS, Email и Fax** из Clients (см. отдельное руководство по коммуникациям).

## Где найти в приложении

1. Войдите как **Admin** или **Project Manager**.
2. Откройте **Clients** → нужный список (Pharmacies, Physicians & Providers, Pharmacists, Medical clinics).
3. В строке клиента с телефоном — действие **Schedule call via SMS** (иконка календаря / meeting invite в меню строки).
4. Заполните форму и нажмите **Send meeting invite**.

**Важно:** у клиента в карточке должен быть **мобильный или рабочий телефон** (для врачей — cell или office phone).

## Как это работает (кратко)

```
PM → FieldWire → SMS клиенту (3 слота)
Клиент → ответ SMS: 1, 2 или 3
FieldWire → событие в Calendar PM + SMS-подтверждение клиенту
```

1. PM выбирает дату, три времени, длительность и тему звонка.
2. Сервер отправляет SMS через **Twilio** на номер из карточки клиента.
3. Клиент отвечает одной цифрой **1**, **2** или **3** с **того же номера**, на который пришло SMS.
4. Система создаёт событие в календаре **отправившего PM** и отправляет клиенту короткое SMS-подтверждение.

Приглашение действует **72 часа**. Новая отправка тому же клиенту отменяет предыдущее неподтверждённое приглашение.

## Пример SMS клиенту

```
FieldWire — meeting request

Hi Dr. Smith,

Jane Doe (Project Manager) would like to schedule a call with you.

Topic: Call with Dr. Smith
Date: Jun 22, 2026 (30 min)

Reply with your preferred time:
1) 10 AM
2) 2 PM
3) 3:30 PM

Text back 1, 2, or 3.
```

## Пример подтверждения клиенту

После ответа **2**:

```
FieldWire — confirmed

Thank you, Dr. Smith!

Call with Dr. Smith
Jun 22, 2026 at 2 PM

— Jane Doe, Project Manager
```

## Инструкция для Project Manager

1. Откройте Clients → выберите клиента **с телефоном**.
2. **Schedule call via SMS**.
3. Проверьте **Meeting title** (по умолчанию «Call with …»).
4. Укажите **Meeting date** и три слота (**Slot 1–3**).
5. При необходимости измените **Duration** (по умолчанию 30 минут).
6. В блоке **SMS preview** проверьте текст — он совпадает с тем, что уйдёт клиенту.
7. **Send meeting invite**.
8. Дождитесь ответа клиента или проверьте статус, снова открыв диалог для этого клиента.
9. После ответа откройте **Calendar** — должно появиться событие с выбранным временем.

## Инструкция для тестирования (заказчик / QA)

### Сценарий A — полный цикл (рекомендуется)

**Подготовка**

- Войти как PM или Admin на https://fieldwire.medicalcontractor.ca
- Выбрать клиента с **реальным мобильным номером**, доступным для теста
- Иметь доступ к этому телефону для чтения и ответа SMS

**Шаги**

| # | Действие | Ожидаемый результат |
|---|----------|---------------------|
| 1 | Clients → Physicians → клиент с телефоном → Schedule call via SMS | Открывается форма с превью SMS |
| 2 | Дата = завтра, слоты 10:00 / 14:00 / 15:30, Send | Сообщение «Meeting invite sent»; SMS на телефон клиента |
| 3 | Прочитать SMS | Видны имя PM, тема, дата, 3 слота, инструкция «Text back 1, 2, or 3» |
| 4 | Ответить SMS: **2** | Приходит SMS-подтверждение с датой и временем слота 2 |
| 5 | Calendar у PM | Новое событие на выбранную дату и время 14:00 (30 мин) |
| 6 | Снова открыть Schedule call via SMS для того же клиента | Статус **Confirmed: slot 2 …** |

### Сценарий B — неверный ответ

| # | Действие | Ожидаемый результат |
|---|----------|---------------------|
| 1 | После инвайта ответить **hello** | SMS: просьба ответить 1, 2 или 3 |
| 2 | Ответить **2** | Подтверждение и событие в календаре |

### Сценарий C — клиент без телефона

| # | Действие | Ожидаемый результат |
|---|----------|---------------------|
| 1 | Клиент без phone/cell | Действие недоступно или ошибка «no phone number on file» |

### Сценарий D — фильтр для поиска клиентов с телефоном

1. Clients → **Filter: Phone** → **Has value**
2. Удобно для массовой проверки списков перед рассылкой

## Связь с другими функциями

| Функция | Связь |
|---------|--------|
| **Clients SMS** | Обычное текстовое SMS; meeting invite — отдельный сценарий с календарём |
| **Calendar** | Подтверждённый слот → личное событие PM |
| **Twilio** | Исходящие SMS и приём ответов клиента (webhook на API) |

## Техническая справка (для IT)

- **Frontend:** диалог Schedule call via SMS в списках Clients.
- **API:** `POST /api/v1/clients/{type}/{id}/send-meeting-invite`, webhook `POST /api/v1/twilio/sms/inbound`.
- **Провайдер SMS:** Twilio (номер отправителя настроен на production).
- **Режим работы:** production — SMS уходят на **реальные номера** из карточек клиентов.
- **Часовой пояс слотов:** America/Toronto (по умолчанию).

## Решение проблем

| Проблема | Что проверить |
|----------|----------------|
| «Failed to send SMS» | Телефон в карточке; баланс Twilio; номер клиента корректен |
| SMS пришло, ответ не сработал | Ответ с **того же номера**, что получил SMS; только **1**, **2** или **3** |
| Нет события в Calendar | Войти под тем же PM, который отправил инвайт; обновить Calendar |
| «No active meeting request» | Прошло более 72 ч или отправлен новый инвайт (старый отменён) |
| Нет кнопки в строке | Роль PM/Admin; у клиента должен быть телефон |
